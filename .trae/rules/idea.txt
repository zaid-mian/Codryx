1. Product Overview

Name: PyGuardian
Type: Python Developer Toolkit (Library + CLI Tool)
Vision:
A full-stack Python code safety & health platform that ensures:

Runtime business logic safety (guards for pre/post conditions)

Dependency and repo health (technical debt, unused code, security, licenses)

Distribution:

Package: PyPI

Code Repo: GitHub

License: MIT

Core Promise: Developers ship safer code faster, with reduced tech debt, and have observable, auditable, actionable feedback.

2. Problem Statement

Python projects today suffer from:

Runtime Logic Bugs: Hardcoded business rules fail silently in production.

Dependency & Code Debt: Unused packages, abandoned libraries, and dead code accumulate.

Security & Compliance Risks: Vulnerable packages, transitive license violations, and insecure code go unnoticed.

Observability Gaps: Failures and technical debt often lack clear reporting and actionable guidance.

Business Logic Safety: Enforce runtime pre/post conditions dynamically.

Dead Code Detection: Identify unused files, functions, and imports.

Security & License Checks: Scan dependencies and transitive libraries for risk.

Developer-Friendly Observability: Logs, dashboards, reports, and CI/CD gates.

Cost & Actionability: Show impact of bad code/dependencies and automate remediation.

5. Core Features
5.1 Runtime Business Logic Guards

Python Decorator: @guard(condition="user.is_active")

Dynamic Rule Loading: Load rules from JSON, YAML, or DB at runtime.

@guard(rules="config/business_rules.json")
def process_order(order):
    ...

Async & Thread-Safe: Works with async def for FastAPI / asyncio apps.

Audit Logging & Tracing: Failures auto-log to structured JSON with context (user ID, request ID).

Zero-Overhead Production Mode: Checks disabled in production via PY_GUARDIAN_MODE=production.

Killer Feature: Visual Rule Graph → static HTML visualizing rule dependencies for onboarding and documentation.

5.2 Dependency & Repo Health Scanner

CLI Commands:

pyguardian deps scan      # Dependency analysis
pyguardian code scan      # Dead code detection
pyguardian security scan  # Vulnerable packages & licenses
pyguardian doctor         # Full health report

Features:

Unused & abandoned packages

Outdated dependencies

Dead code detection (files/functions/imports)

Transitive license risk detection

Cost estimation (impact on build time, cloud costs)

Automated migration PRs for GitHub (removes unused deps and updates imports)

Health Score Dashboard (0–100)

CI/CD Gatekeeper → fail build if score < threshold

Killer Feature: Automated Migration PR → eliminates manual dependency cleanup, huge productivity win.

5.3 Combined CLI + Python API

CLI:

pyguardian deps scan
pyguardian code scan
pyguardian security scan
pyguardian doctor
pyguardian help

Python API:

from pyguardian import guard, scan_dependencies, scan_code, enable_strict_mode

@guard("user.is_active")
def create_order(user):
    ...

enable_strict_mode()   # All checks in dev
scan_dependencies()    # Programmatic usage
scan_code()
